# 执行计划

## 概述

加载计划，批判性地复审，执行所有任务，完成后报告。

**开始时声明：** "我正在使用 executingPlan 技能来实现这份计划。"

## 流程

### 第一步：加载并复审计划
1. 阅读计划文件
2. 批判性复审 —— 识别计划中任何疑问或顾虑
3. 如果有顾虑：在开始之前向你的伙伴提出
4. 如果没有顾虑：创建 TodoWrite 并继续

### 第二步：执行任务

对每个任务：
1. 标记为进行中
2. 严格按照每一步执行（计划已拆分为小步）
3. 按规定运行验证
4. 标记为已完成

### 第三步：完成开发

所有任务都完成并通过验证后：
- 声明："我正在使用 code-review 技能来完成这项工作。"
- **必需子技能：** 使用 code-review 技能来复审本任务创建和修改的所有代码。
- 遵循该技能来验证测试、呈现选项、执行选择

## 何时停止并寻求帮助

**遇到以下情况立即停止执行：**

- 遇到阻碍（缺少依赖、测试失败、指令不清）
- 计划存在阻止开始的严重漏洞
- 你不理解某条指令
- 验证反复失败

**询问澄清而不是猜测。**

## 何时回到前面的步骤

**在以下情况回到复审（第一步）：**
- 伙伴根据你的反馈更新了计划
- 根本方法需要重新思考

**不要硬闯障碍** —— 停下来询问。

## 牢记
- 先批判性地复审计划
- 严格按照计划步骤执行
- 不要跳过验证
- 当计划要求时引用相关技能
- 遇到阻碍时停下，不要猜测
- 如果用户没有明确声明使用 Git 管理当前任务，则在任务完全完成之前不要执行任何 Git 操作
