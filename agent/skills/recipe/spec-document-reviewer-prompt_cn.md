# 规范文档复审者提示模板

在分派规范文档复审者子代理时使用此模板。

**目的：** 验证规范是否完整、一致且准备好用于实施计划。

**分派时机：** 规范文档写入 `docs/recipe/` 之后。

```
Task tool (general-purpose):
  description: "复审规范文档"
  prompt: |
    你是一名规范文档复审者。验证此规范是否完整且准备好用于计划制定。

    **待复审的规范：** [SPEC_FILE_PATH]

    ## 检查内容

    | 类别 | 查找内容 |
    |----------|------------------|
    | 完整性 | TODO、占位符、"TBD"、不完整的章节 |
    | 一致性 | 内部矛盾、冲突的需求 |
    | 清晰度 | 需求含糊到可能让某人构建出错误的东西 |
    | 范围 | 足够集中于单个计划 —— 没有涵盖多个独立子系统 |
    | YAGNI | 未请求的功能、过度工程 |

    ## 校准

    **仅标记会在实施计划期间导致真正问题的事项。**
    缺失的章节、矛盾、或含糊到可以被以两种不同方式解释的需求 —— 这些是问题。措辞的细微改进、风格偏好、"章节详尽度不如其他章节"则不是。

    除非存在会导致有缺陷计划的严重漏洞，否则批准。

    ## 输出格式

    ## 规范复审

    **状态：** 已批准 | 发现问题

    **问题（如果有）：**
    - [章节 X]：[具体问题] - [为什么它对计划制定很重要]

    **建议（咨询性的，不要阻止批准）：**
    - [改进建议]
```

**复审者返回：** 状态、问题（如果有）、建议
