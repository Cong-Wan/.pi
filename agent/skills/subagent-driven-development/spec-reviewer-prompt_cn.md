# 规范合规性复审者提示模板

在分派规范合规性复审者子代理时使用此模板。

**目的：** 验证实现者构建了所请求的内容（既不多也不少）

```
Task tool (general-purpose):
  description: "复审 Task N 的规范合规性"
  prompt: |
    你正在复审实现是否匹配其规范。

    ## 请求了什么

    [任务需求的完整文本]

    ## 实现者声称他们构建了什么

    [来自实现者的报告]

    ## 关键：不要相信报告

    实现者可疑地快速完成。他们的报告可能不完整、不准确或过于乐观。你必须独立验证一切。

    **不要：**
    - 相信他们关于实现了什么的说法
    - 相信他们关于完整性的说法
    - 接受他们对需求的解释

    **要：**
    - 阅读他们编写的实际代码
    - 逐行比较实际实现与需求
    - 检查他们声称实现但实际未实现的缺失部分
    - 查找他们未提及的额外功能

    ## 你的工作

    阅读实现代码并验证：

    **缺失的需求：**
    - 他们是否实现了所有被请求的内容？
    - 是否存在他们跳过或遗漏的需求？
    - 他们是否声称某些东西有效但实际未实现？

    **额外/不需要的工作：**
    - 他们是否构建了未被请求的内容？
    - 他们是否过度工程化或添加了不必要的功能？
    - 他们是否添加了规范中没有的"锦上添花"？

    **误解：**
    - 他们是否以不同于预期的方式解释需求？
    - 他们是否解决了错误的问题？
    - 他们是否实现了正确的功能但方式错误？

    **通过阅读代码验证，而非相信报告。**

    报告：
    - ✅ 规范合规（如果代码检查后一切都匹配）
    - ❌ 发现问题：[具体列出缺失或多余的内容，带 file:line 引用]
```
